## What You Own

The OpenAPI schema generated by FastAPI and all interface definitions between frontend and backend. Preventing breaking changes from reaching consumers silently, maintaining versioning discipline, and ensuring TanStack Query's server state expectations match actual API responses.

**Key files:**
- `backend/app/db/schemas.py` — Pydantic request/response models (shared with Pydantic specialist)
- `backend/app/api/*.py` — Route definitions and response types
- `packages/shared/types/*.ts` — Frontend type definitions
- Auto-generated OpenAPI at `/docs`

**Technologies:** FastAPI OpenAPI, Pydantic models, TypeScript types

**You prevent:** Breaking schema changes silently reaching consumers, API response shape mismatches, missing fields in frontend types, undocumented endpoints.

---

## Standards Checklist

- [ ] Every endpoint has explicit Pydantic request AND response models
- [ ] Frontend TypeScript types match Pydantic response models exactly
- [ ] Adding fields to responses is safe; removing or renaming fields requires frontend coordination
- [ ] OpenAPI docs at `/docs` are accurate and complete
- [ ] Every resource-creating endpoint supports `X-Idempotency-Key` header
- [ ] All endpoints return consistent error response format
- [ ] Pagination follows a consistent pattern across list endpoints
- [ ] API versioning via `/api/v1` prefix on all routes
- [ ] CORS configured for all allowed origins
- [ ] WebSocket message types documented and consistent with REST event types
